## 四、Headless全功能测试方案

### 基础设施
- 复用已有: ecos-tests/data-workbench-smoke.mjs (Playwright headless)
- 扩展为: ecos-tests/ecos-full-regression.mjs
- 前置: 后端(:8080)+前端(:3000)均启动
- 执行: node ecos-tests/ecos-full-regression.mjs
- 输出: 测试报告(console+JSON文件)

### 测试矩阵（每周递增）
W1: 15核心API端点 + 5页面可达(登录+首页+数据工作台+本体+AI)
W2: +4引擎health/config/status/tasks + 前端4引擎页
W3: +数据工作台9Tab API + 本体CRUD API
W4: +知识6Tab API + AI Agent/Guardrails API + 安全脱敏API
W5: +系统管理CRUD + 项目工作台页面
W6: **全量50+API端点 + 15+页面可达性 + 场景E2E** → 完整测试报告

### 测试报告格式
每轮输出包含:
1. 通过/失败/总计统计
2. 每个端点: 方法+路径+状态码+响应时间
3. 每个页面: URL+标题+截图路径
4. 回归对比: 新增失败项标红
5. 终轮(W6)输出Markdown报告: ecos-tests/reports/regression-W6.md
