## Risks
- The `agent_cost` partitioned table in PG uses `VARCHAR(64)` as part of composite PK with `created_at`. This is correct but means the id must be unique per created_at value.
- MySQL `outbox_event` has `BIGINT AUTO_INCREMENT` as part of composite PK with `created_at` — MySQL supports this for partitioned tables.
- Oracle DDL uses PL/SQL exception blocks extensively for `IF NOT EXISTS` — these are more verbose but functionally correct.
