### 4.1 Doris Runner 技术选型

| 维度 | DuckDB (旧) | Apache Doris (新) |
|------|-----------|------------------|
| 部署方式 | 嵌入式 JDBC，文件数据库 | **Docker 独立服务**，1FE + 1BE |
| JDBC 接入 | `duckdb_jdbc` | **MySQL 协议** — 标准 `mysql-connector-j` |
| 端口 | 无独立端口 | FE HTTP :8030, FE MySQL :9030, BE :8040 |
| 存储 | 本地 `*.duckdb` 文件 | BE 节点本地存储，支持多副本 |
| 查询能力 | 单机 OLAP | 分布式 MPP，兼容 MySQL 函数 |
| 适用场景 | 开发/轻量分析 | **生产级** T1 管道任务（批量 ETL + 聚合） |

### 4.2 Doris Docker Compose（开发环境）

```yaml
# docker-compose-doris.yml
version: "3"
services:
  doris-fe:
    image: apache/doris:3.0.3-fe-x86_64
    container_name: ecos-doris-fe
    ports:
      - "8030:8030"   # HTTP
      - "9030:9030"   # MySQL protocol (JDBC 用这个)
    environment:
      - FE_SERVERS=fe1:172.20.0.2:9010
      - BE_SERVERS=be1:172.20.0.3:9050
    networks:
      ecos-net:
        ipv4_address: 172.20.0.2
    volumes:
      - doris-fe-data:/opt/apache-doris/fe/doris-meta

  doris-be:
    image: apache/doris:3.0.3-be-x86_64
    container_name: ecos-doris-be
    ports:
      - "8040:8040"
    environment:
      - FE_SERVERS=fe1:172.20.0.2:9010
      - BE_ADDR=172.20.0.3:9050
    networks:
      ecos-net:
        ipv4_address: 172.20.0.3
    volumes:
      - doris-be-data:/opt/apache-doris/be/storage

volumes:
  doris-fe-data:
  doris-be-data:

networks:
  ecos-net:
    ipam:
      config:
        - subnet: 172.20.0.0/16
```

> **JDBC 连接串**: `jdbc:mysql://172.20.0.2:9030`（FE MySQL 协议端口）
> **Spring Boot 配置**: `spring.datasource.doris.url=jdbc:mysql://172.20.0.2:9030`
> **注意**: 开发环境 1FE+1BE 即可，生产环境建议 3FE+3BE。

### 4.4 NSGA-II 帕累托优化器（P3-3 补做）

> **来源**: Phase 3 看板标记 DONE 但零代码 — Sprint 5 补做。
> **定位**: 认知引擎第三推理器（Rule Engine → Causal Reasoner → **Pareto Optimizer**）

| 维度 | 设计 |
|------|------|
| 算法 | NSGA-II（非支配排序遗传算法）— 自研轻量实现 |
| 输入 | 多目标优化问题：决策变量 + 约束 + 目标函数（min/max） |
| 输出 | **帕累托前沿** — 非支配解集 |
| 部署 | 嵌入 `databridge-cognitive` 模块，零外部依赖 |
| API | `POST /api/v1/cognitive/optimize` |
| 典型场景 | 数据管道 Q3 成本 ↔ 时效 ↔ 质量 三维权衡 |

**实现要点**（纯 Java，~300行）：
```java
// NsgaIIOptimizer.java — 认知引擎帕累托优化器
@Service
public class NsgaIIOptimizer {
    // 1. 种群初始化（随机生成 N 个可行解）
    // 2. 非支配排序（Fast Non-Dominated Sort）
    // 3. 拥挤度距离计算（Crowding Distance）
    // 4. 锦标赛选择 + 交叉 + 变异 → 下一代
    // 5. 迭代至收敛 → 返回 Pareto Front
    public ParetoFront optimize(MultiObjectiveProblem problem) { ... }
}
```

**数据结构**:
```java
// 多目标问题定义
record MultiObjectiveProblem(
    List<DecisionVariable> variables,  // 决策变量 + 范围
    List<Constraint> constraints,       // 约束
    List<Objective> objectives          // 目标函数 (MIN/MAX)
) {}

// 帕累托前沿
record ParetoFront(
    List<ParetoSolution> solutions,     // 非支配解
    int generations,                    // 迭代代数
    long elapsedMs                      // 耗时
) {}
```

**Maven 依赖**: 零。纯 Java 标准库实现（List/Map/Random/Comparator）。

**前端对接**: CognitiveOperatingSystem 页面新增 `ParetoOptimizer` 卡片，调用 `POST /api/v1/cognitive/optimize` 展示帕累托前沿散点图。

### 4.5 认知引擎推理链路

```
认知引擎 (databridge-cognitive) — 三推理器串联，同模块，同进程

┌─────────────────────────────────────────────────────────────┐
│                    CognitiveController                       │
│         /api/v1/cognitive/{blueprint|reason|optimize|plan}  │
└──────────────────────────┬──────────────────────────────────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
         RuleEngine   CausalReasoner  NsgaIIOptimizer
        (规则匹配)    (因果链推理)    (帕累托优化)
              │            │            │
              └────────────┼────────────┘
                           ▼
                   ExecutionPlan
                           │
                           ▼
              Hermes Agent Runtime (执行)
```

| 推理器 | 输入 | 输出 | 算法 |
|--------|------|------|------|
| **RuleEngine** | 事实(Facts) + 规则(Rules) | 匹配的动作列表 | Rete 模式匹配 |
| **CausalReasoner** | 事件 + 因果图(KG) | 因果路径 + 根因 | 图遍历 + 阈值剪枝 |
| **NsgaIIOptimizer** | 多目标问题(变量+约束+目标) | 帕累托前沿解集 | 非支配排序 + 遗传进化 |

> **关键**: 三个推理器都是 `databridge-cognitive` 模块的**内部组件**，不独立部署。`CognitiveController` 是唯一对外入口，编排推理链路，产出 `ExecutionPlan` 下发给 Hermes 执行。
