# 4. Entity Meta Model

---

## 定义

Entity 描述业务对象类型。

例如：

```text
Customer

Order

Facility

Asset

Employee

Case

Goal
```

---

## 元模型

```java
public class EntityDefinition {

    UUID id;

    String code;

    String name;

    String description;

    EntityCategory category;

    LifecycleDefinition lifecycle;

}
```

---

## EntityCategory

```java
MASTER

TRANSACTION

EVENT

REFERENCE

ANALYTICS
```

---

## 示例

```yaml
entity:

  code: Customer

  name: 客户

  category: MASTER
```

---

# 5. Property Meta Model

---

## 元模型

```java
public class PropertyDefinition {

    UUID id;

    String code;

    String name;

    DataType type;

    Boolean required;

    Boolean indexed;

    ValidationRule validation;
}
```

---

## DataType

```java
STRING

TEXT

INTEGER

LONG

DECIMAL

BOOLEAN

DATE

DATETIME

ENUM

OBJECT

ARRAY
```

---

## 示例

```yaml
properties:

 - code: customerName
   type: STRING

 - code: annualRevenue
   type: DECIMAL
```

# 6. Relationship Meta Model

---

关系是Foundry最重要能力。

---

## 元模型

```java
public class RelationshipDefinition {

    UUID id;

    EntityCode source;

    EntityCode target;

    RelationshipType type;

    Cardinality cardinality;
}
```

---

## RelationshipType

```java
OWNS

USES

BELONGS_TO

REPORTS_TO

SUPPORTS

IMPACTS

RELATED_TO
```

---

## Cardinality

```java
ONE_TO_ONE

ONE_TO_MANY

MANY_TO_ONE

MANY_TO_MANY
```

---

## 示例

```yaml
Customer

  OWNS

Facility
```

# 7. Metric Meta Model

# 12. Ontology DSL

---

ECOS提供DSL。

---

## Customer

```yaml

  code: Customer

  properties:

    - code: name
      type: STRING

    - code: revenue
      type: DECIMAL
```

---

## Relationship

  source: Customer

  target: Facility

  type: OWNS

# 14. Object Runtime Mapping

---

Ontology：

```yaml
Customer
```

自动生成：

```java
ObjectType:

Customer
```

---

实例：

```json
{
  "id":"cust001",
  "type":"Customer",
  "properties":{
      "name":"ACME"
  }
}
```
