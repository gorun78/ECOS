## 一、ECOS 使用对象分析

### 1.1 角色全景图

ECOS 作为企业级认知操作系统（DIKW架构），使用对象覆盖从决策层到技术层的全角色体系：

| 层级 | 角色 | 职责 | 典型使用场景 | 使用频率 |
|------|------|------|-------------|:--------:|
| **决策层** | 公司管理层/领导 | 经营数据洞察、战略决策 | 经营看板（营收/利润/回款/两金压降指标）、World Model What-If分析 | 每日 |
| **管理层** | 业务负责人 (Board Admin) | 流程定义、看板创建、数据标准制定 | Ontology Designer建模、Workflow Designer拖拽流程、看板管理 | 每周 |
| **管理层** | 部门经理 | 部门经营数据、人员管理、审批 | 产值分配审核、回款认领确认、部门看板 | 每日 |
| **执行层** | 业务执行者 (Board Member) | 日常业务操作、数据录入 | 项目跟踪登记、合同录入、计量/开票/回款登记 | 每天高频 |
| **执行层** | 普通用户 (Viewer) | 只读查看 | 查看经营看板、浏览数据资产 | 按需 |
| **财务层** | 财务人员 | 财务目标设定、成本数据、财税管理 | 营业收入/利润总额/回款目标设定、人力成本填报、税务筹划 | 月度/季度 |
| **技术层** | 系统管理员 (Sys Admin) | 全局资源配置、用户/机构/角色/权限管理 | IAM管理、系统参数配置、安全审计 | 按需 |
| **技术层** | 数据工程师 | 数据管道设计、质量规则配置 | Pipeline Designer、Data Quality规则引擎、ETL任务 | 每周 |
| **技术层** | AI/Agent 开发者 | Agent创建/测试/发布 | Agent Builder配置Prompt/Tools/Knowledge | 按需 |
| **外部层** | 合作伙伴/供应商 | 外部协作 | 供应商门户（未来）、数据共享 | 按需 |
| **监管层** | 审计/合规人员 | 审计追踪、合规检查 | 操作审计日志、数据安全合规审查 | 按需 |

## 二、系统运行需要的系统级配置项

### 2.1 当前已有配置（application.yml）

基于 `sysman-boot/src/main/resources/application.yml` 分析：

| 配置项 | 当前值 | 说明 | 产品化改进建议 |
|--------|--------|------|---------------|
| `server.port` | `8081` | 服务端口 | 通过环境变量注入 `SERVER_PORT` |
| `spring.datasource.url` | `jdbc:postgresql://localhost:5432/sys_man` | 数据库连接 | 提取为 `DB_URL`/`DB_USER`/`DB_PASS` 环境变量 |
| `spring.datasource.username` | `root` | DB用户 | ⚠️ 不应硬编码，需环境变量 |
| `spring.datasource.password` | `root` | DB密码 | ⚠️ 不应硬编码，需环境变量/密钥管理 |
| `spring.datasource.hikari.maximum-pool-size` | `20` | 连接池大小 | 可配置 |
| `auth.enabled` | `true` | 认证开关 | 生产必须为 true |
| `auth.whitelist.paths` | 5个路径 | 白名单 | 需按环境差异化 |
| `llm.deepseek.api-key` | `${DEEPSEEK_API_KEY:}` | LLM API Key | 环境变量注入 ✅ |
| `llm.deepseek.model` | `deepseek-chat` | LLM 模型 | 可配置 |
| `jwt.private-key` | PEM格式RSA私钥 | JWT签名密钥 | ⚠️ 私钥硬编码在配置文件中，风险极高 |
| `jwt.public-key` | PEM格式RSA公钥 | JWT验证密钥 | 应在启动时从密钥服务获取 |
| `hermes.engine.*` | Hermes引擎配置 | Agent引擎参数 | 可配置 |
| `logging.level.*` | INFO/DEBUG | 日志级别 | 按环境配置 |

### 2.3 配置管理策略建议

```
┌─────────────────────────────────────────────────────────┐
│ 优先级：环境变量 > 配置文件 > 数据库配置表 > 默认值        │
├─────────────────────────────────────────────────────────┤
│ Layer 1: 敏感信息 → 环境变量/密钥管理服务 (Vault/KMS)     │
│   - JWT私钥、DB密码、LLM API Key、企业微信Secret          │
│                                                          │
│ Layer 2: 环境差异 → application-{profile}.yml             │
│   - 数据库URL、日志级别、CORS白名单、缓存类型              │
│                                                          │
│ Layer 3: 运行时可调 → 数据库配置表 ecos_sys_config         │
│   - 功能开关、阈值参数、通知模板、UI偏好                   │
│                                                          │
│ Layer 4: 硬编码默认 → 源码 default 值                      │
│   - 最后兜底，确保不配置也能启动                           │
└─────────────────────────────────────────────────────────┘
```

### 5.2 第一优先级行动项（P0）

1. **改造 AuthController**: 登录查数据库，接入 BCrypt 密码验证

3. **前端登录页**: 用户名+密码登录，对接后端 /auth/login
4. **前端路由守卫**: Token 过期自动刷新，未登录跳转
5. **前端菜单控制**: 按角色显示/隐藏菜单项
