
# 8.2 Action Runtime

---

## 功能目标

执行Ontology中定义的Action。

---

示例：

Customer对象

支持：

```text
Approve

Reject

Suspend

Activate
```

---

Order对象

支持：

```text
Submit

Pay

Ship

Complete

Cancel
```

---

## Action执行架构

```text
User

↓

Action Runtime

↓

Rule Engine

↓

Policy Engine

↓

State Machine

↓

Object Update

↓

Event Publish

↓

Workflow Trigger
```

---

# 8.3 Action定义模型

---

## Action结构

```json
{
  "code":"ApproveCustomer",
  "name":"审批客户",
  "target":"Customer",
  "type":"StateTransition"
}
```

---

字段：

| 字段     | 说明   |
| ------ | ---- |
| Code   | 动作编码 |
| Name   | 动作名称 |
| Target | 目标对象 |
| Type   | 动作类型 |
| Rule   | 规则   |
| Policy | 权限策略 |

---

# 8.4 Action分类

---

## State Action

状态变更。

示例：

```text
Approve

Reject

Suspend
```

---

## Data Action

数据修改。

示例：

```text
UpdateLevel

UpdateOwner
```

---

## Workflow Action

启动流程。

示例：

```text
StartApprovalFlow
```

---

## Agent Action

调用Agent。

示例：

```text
RiskAssessment
```

---

## Integration Action

调用外部系统。

示例：

```text
CreateERPOrder
```

---

# 8.5 Rule Engine

---

## 功能目标

统一规则执行平台。

---

支持：

```text
业务规则

校验规则

决策规则

安全规则

Agent规则
```

---

## Rule结构

```json
{
  "code":"CustomerLevelRule",
  "expression":"sales_amount > 10000000"
}
```

---

# 8.6 Rule分类

---

## Validation Rule

校验规则

---

示例：

```text
手机号不能为空
```

---

表达式：

```sql
phone IS NOT NULL
```

---

## Calculation Rule

计算规则

---

示例：

```text
客户评分
```

---

公式：

```text
销售额 × 0.6

+

利润 × 0.4
```

---

## Decision Rule

决策规则

---

示例：

```text
信用等级判断
```

---

## Event Rule

事件触发规则

---

示例：

```text
订单金额 > 100万

↓

通知财务总监
```

---

---
