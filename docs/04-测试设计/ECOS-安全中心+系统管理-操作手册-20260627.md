## 一、安全中心操作

### 1.1 查看/修改安全配置

安全配置控制密码策略、MFA、会话超时等全局安全参数。

**查看当前配置**:
```bash
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/security/profile
```

**查看所有配置模板**:
```bash
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/security/profiles
```

**修改安全配置**:
```bash
curl -X PUT \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"passwordMinLength": 12, "mfaEnabled": true, "sessionTimeout": 1800}' \
  http://localhost:8080/api/v1/security/profile
```

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|:--:|
| `passwordMinLength` | int | 密码最小长度 | 8 |
| `mfaEnabled` | bool | 是否启用多因素认证 | false |
| `sessionTimeout` | int | 会话超时(秒) | 3600 |
| `maxLoginAttempts` | int | 最大登录尝试次数 | 5 |
| `lockoutDuration` | int | 锁定时间(秒) | 900 |

### 1.3 ABAC 策略管理

ABAC (Attribute-Based Access Control) 属性级访问控制，基于用户/资源/环境属性动态授权。

**查看 ABAC 策略列表**:
```bash
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/abac/policies
```

**创建 ABAC 策略**:
```bash
curl -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "财务数据只读",
    "description": "财务部门员工只读访问财务报表",
    "rules": [{
      "effect": "ALLOW",
      "condition": {
        "user.dept": "FINANCE",
        "resource.type": "FINANCIAL_REPORT",
        "action": "READ"
      }
    }]
  }' \
  http://localhost:8080/api/v1/abac/policies
```

**查看策略详情**:
```bash
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/abac/policies/{policyId}
```

**更新策略**:
```bash
curl -X PUT \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "财务数据只读-v2", "rules": [...]}' \
  http://localhost:8080/api/v1/abac/policies/{policyId}
```

**删除策略**:
```bash
curl -X DELETE \
  -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/abac/policies/{policyId}
```

## 三、告警处置 SOP

### 3.1 安全告警 — 登录失败暴破

**症状**: 同一用户 5 分钟内登录失败 > 5 次

**处置步骤**:
1. 查询审计日志确认攻击来源 IP:
   ```bash
   curl -H "Authorization: Bearer $TOKEN" \
     "http://localhost:8080/api/v1/audit/logs?eventType=LOGIN_FAILED&limit=100"
   ```
2. 临时锁定账户:
   ```bash
   curl -X PUT \
     -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"status": "LOCKED"}' \
     http://localhost:8080/api/v1/system/users/{userId}
   ```
3. 通知用户修改密码
4. 解锁账户 (确认安全后):
   ```bash
   curl -X PUT \
     -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"status": "ACTIVE"}' \
     http://localhost:8080/api/v1/system/users/{userId}
   ```

### 3.2 安全告警 — 权限变更

**症状**: 角色/权限被非预期修改

**处置步骤**:
1. 查询审计日志定位操作人:
   ```bash
   curl -H "Authorization: Bearer $TOKEN" \
     "http://localhost:8080/api/v1/audit/logs?resource=ROLE&action=UPDATE&limit=50"
   ```
2. 回滚角色权限:
   ```bash
   curl -X PUT \
     -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"permissionIds": ["perm_read", "perm_write"]}' \
     http://localhost:8080/api/v1/system/roles/{roleId}
   ```
3. 更新安全配置，收紧变更审批:
   ```bash
   curl -X PUT \
     -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"requireApprovalForRoleChange": true}' \
     http://localhost:8080/api/v1/security/profile
   ```

### 3.3 系统告警 — 数据库连接池耗尽

**症状**: API 返回 500，日志含 `Connection pool exhausted`

**处置步骤**:
1. 检查监控仪表盘:
   ```bash
   curl -H "Authorization: Bearer $TOKEN" \
     http://localhost:8080/api/v1/ecos/monitoring/dashboard
   ```
2. 临时调大连接池:
   ```bash
   curl -X PUT \
     -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"configValue": "50"}' \
     http://localhost:8080/api/v1/system/config/db.pool.maxSize
   ```
3. 排查慢查询，重启受影响服务
