## 三、各角色关键发现

### 🏗️ ECOS-ARCH — 对后端代码的深度扫描

| 发现 | 严重度 |
|------|:------:|
| 🚨 **4个Controller全部用ConcurrentHashMap（内存）**，重启即丢 | 🔴 高 |
| 🚨 **IAM两套认证**：AuthController用UUID Token，AuthServiceImpl已有完整JWT实现 | 🔴 高 |
| 🚨 **ObjectController用字符串拼接SQL**，存在注入风险 | 🔴 高 |
| ✅ Workflow Designer 后端骨架就绪 | 🟢 低 |
| ✅ Ontology Designer 前端已基本完成，后端只需持久化迁移 | 🟢 低 |

**推荐架构方案（全体ARCH+BE一致）**：
- Object Runtime：JSONB三表架构（ecos_object + ecos_object_data + ecos_object_relation）
- 状态机引擎：复用已有 Workflow Engine，不重复造轮
- Ontology：单表JSONB方案（全量加载→编辑→全量保存）
- IAM：统一JWT（已有jjwt依赖0.12.5，已有TokenServiceImpl）
