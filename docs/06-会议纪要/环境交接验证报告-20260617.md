## 三、模式分析

### 3.1 正常端点特征
- `GlossaryController`, `WorldModelController`, `DqController` — 这三个是 Sprint 0 明确完成 DB 持久化迁移的
- 它们共同特征：从 `ConcurrentHashMap` 迁移到了 `JdbcTemplate`，查询轻量（单表 SELECT）

### 3.2 超时端点特征
- 所有超时端点**都不是** Sprint 0 中迁移到 DB 的模块
- `Workflow` 虽然注释说"PostgreSQL 持久化"，`WorkflowService` 注入了 `WorkflowRepository`，但实际挂死
- **可能原因：**
  1. **Spring Security 拦截** — SecurityConfig 白名单极窄（仅 `/auth/**`、`/api/health`、`/agent-mesh/**`），其余需认证
  2. **但安全配置未完全生效** — Glossary/WorldModel/DQ 无 Token 仍返回 200，说明认证过滤器有漏洞
  3. **Bean 初始化死锁 / 循环依赖** — 某些 Service 注入的 Repository/Engine 可能未正确初始化
  4. **外部依赖不可用** — KnowledgeGraph 可能依赖 Neo4j，AgentMesh 可能依赖 Agent Runtime

### 3.3 404 端点特征
- `ObjectController` — 需要 `entityCode` 参数，不带参数返回 404（快速）
- `OntologyController` — 需要 `ontologyId`，不带参数返回 404（快速）
- 这些端点至少**响应迅速**，说明 Controller Bean 正常加载
