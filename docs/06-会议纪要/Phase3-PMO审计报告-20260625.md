## 执行策略

### 并行窗口

由于后端只有 1 个 Gateway（单 Tomcat 8080），BE 任务不能真正并行。
采用**串行冲刺**：每完成一个 P3-x，编译安装→重启 Gateway→验证→继续下一个。

```
Week 1-2 (P3-1): DuckDB + MinIO 数据湖         → 5 天
Week 2-3 (P3-2): Workbook 沙盒                  → 7 天（含 Python 探测）
Week 3-4 (P3-3): NSGA-II 帕累托                 → 5 天
Week 4-5 (P3-4): MQTT/OPC-UA 数字孪生           → 7 天
Week 5-6 (P3-5): OpenTelemetry 全链路            → 5 天
Week 6   (P3-6): 多租户隔离+计费                 → 5 天
Week 6   最后 : 全量回归验证                     → 2 天
```

### 每个子项目的标准流程

```
BE 执行 → mvn install + 重启 Gateway → curl 验证 API 200
  → FE 执行 → tsc + vite build → 页面渲染
  → QA 验证 → 全链路 curl
```
