## 四、环境陷阱（PMO 必须知道，否则会翻车）

### 🪤 陷阱1：`~` 路径被 Hermes 重定向
在 terminal 命令中，`~` 展开为 `/home/guorongxiao/.hermes/profiles/gorunkol/home/`，**不是** `/home/guorongxiao/`。
**所有涉及文件路径的命令必须用绝对路径 `/home/guorongxiao/...`。**

### 🪤 陷阱2：只认这个 JDK
```bash
export JAVA_HOME=/home/guorongxiao/.local/jdk/jdk-17.0.19+10
```
其他 Java（包括 `which java` 找到的 wrapper）会导致 UNC 路径双写 bug。

### 🪤 陷阱3：先 install 再 spring-boot:run
错误做法：`mvn spring-boot:run -pl sysman-boot -am`（-am 会把 run 传到父 POM）
正确做法：先 `mvn install -DskipTests`，再 `mvn spring-boot:run -pl sysman-boot`

### 🪤 陷阱4：两个 GlobalExceptionHandler（已修复）
存在两个同名 bean：`sysman-impl/.../controller/GlobalExceptionHandler`（旧）和 `sysman-boot/.../handler/GlobalExceptionHandler`（新）。
**已删除旧的（impl 中的）**，保留 boot 中的新版本。PMO 不要再创建回去。

### 🪤 陷阱5：看板 DONE ≠ 代码 DONE
总看板显示 17/17 全部 DONE，但实际代码经历了一次大迁移+环境修复，**不要信任看板的 DONE 状态**。
验证方法：`curl http://localhost:8081/sys-man/api/health` 确认运行，然后逐个验证关键 API。
