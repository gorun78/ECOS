### P3-1: 嵌入式数据湖（DuckDB + MinIO）

**可行性判定**: ✅ **可行**

#### 2.1.1 DuckDB JDBC 与 PG JDBC 共存验证

- **结论**: 可共存，无冲突风险。
- **原理**: DuckDB 使用 `org.duckdb:duckdb_jdbc` 驱动，连接 URL 为 `jdbc:duckdb:./data/ecos_olap.db`（嵌入式文件数据库）；PostgreSQL 使用 `org.postgresql.Driver`，连接 URL 为 `jdbc:postgresql://localhost:5432/...`。两者通过不同的 `DataSource` Bean 管理，互不干扰。
- **当前 PG 连接方式**: 网关模块通过 `JdbcTemplate` 直连 PG（见 AlertController.java line 22），DuckDB 应创建独立的 `@Bean("duckdbJdbcTemplate")`。
- **建议**: 在 `databridge-gateway` 的配置类中创建独立的 `DuckDBConfig`，使用 `@Qualifier` 区分 DataSource。

#### 2.1.2 MinIO 本地二进制 WSL 兼容性

- **结论**: 可行，WSL2 完全支持 Linux AMD64 二进制。
- **部署方案**: MinIO 官方提供 `minio` 和 `mc` 两个 Linux AMD64 静态编译二进制，WSL Ubuntu 可直接运行。
- **降级路径**: 若 MinIO 二进制异常 → 方案已内置降级为本地文件系统（DuckDB 直接读本地 Parquet 文件）。

#### 2.1.3 Parquet 导出可行性

- **结论**: 可行。使用 DuckDB JDBC 的 `COPY ... TO 's3://...' (FORMAT PARQUET)` 或 Apache Parquet 库。
- **补充建议**: 可考虑引入 `org.apache.parquet:parquet-avro` 或直接用 DuckDB 的导出功能（最简方案）。

#### 2.1.4 模块归属

| 类 | 归属模块 | 理由 |
|----|---------|------|
| `DuckDBQueryService.java` | `databridge-gateway` | 网关层统一提供数据查询服务 |

| `minio-setup.sh` | `databridge-v2/scripts/` | 运维脚本 |

### P3-2: 多语言 Workbook 沙盒

**可行性判定**: ✅ 可行（R Runtime 需标注为"探测性功能"）

#### 2.2.1 Python subprocess 隔离安全性

- **结论**: 基本安全，但非沙盒级隔离。
- **当前约束**:
  - `ProcessBuilder` + `waitFor(30, TimeUnit.SECONDS)` — 超时控制
  - `p.destroyForcibly()` — 超时强杀
  - 计划限制输出 ≤10MB、行数 ≤1000
- **缺失的安全措施**:
  - 无系统级资源限制（`ulimit` 在 WSL 中可用但需在 `ProcessBuilder` 中显式设置）
  - 无网络隔离（subprocess 可访问 localhost 所有端口）
  - 无文件系统隔离（subprocess 可读 `data/workbooks/` 之外的文件）
- **建议**: 
  - 设置 `ulimit -v 524288`（512MB 虚拟内存）限制内存
  - 工作目录使用 `Path.of("data/workbooks", sessionId)`，限制文件操作在该目录下
  - 文档明确标注"进程级隔离，非容器沙盒"

#### 2.2.2 R Runtime 可行性

- **结论**: ❌ 当前不可用 — `Rscript` 未安装。
- **实际环境**: `Rscript: command not found`
- **建议**: 
  - P3-2.4 任务在开发时直接实现友好降级：调用前探测 `which Rscript`，不存在时返回 `{"code":200,"data":{"available":false,"message":"R 运行时未安装，请执行 sudo apt install r-base"}}`
  - 不在验收标准中强依赖 R 执行成功

#### 2.2.3 SQL Runtime

- **结论**: 可行。复用现有 `JdbcTemplate` 连接池，执行 SELECT/DML。
- **安全注意事项**: 
  - 禁止 DDL（DROP/ALTER/CREATE）— 语句前缀白名单检查
  - 禁止多语句（`;` 分隔）— 防止 SQL 注入
  - 使用 `JdbcTemplate.query()` 而非 `execute()` 限制为只读查询（DML 需求需单独评估）

#### 2.2.4 模块归属
