# 1. 微服务设计原则

ECOS采用：

```text
Domain Driven Microservice
Event Driven Architecture
API First
Database Per Service
```

原则：

```text
一个Bounded Context
=
多个微服务

一个微服务
=
一个独立数据库Schema
```

禁止：

```text
共享数据库

跨服务写库

跨服务事务
```

采用：

```text
Saga

Event Choreography

Outbox Pattern
```

# 12. 服务依赖矩阵

```text

    ↓

catalog
    ↓

ontology
    ↓

object
    ↓

workflow
    ↓

agent
    ↓

knowledge
    ↓

mission
```

---

```text
Mission → Identity

Agent → Workflow → Agent
```
