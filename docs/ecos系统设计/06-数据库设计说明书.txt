# 1. 数据库总体设计

ECOS采用 Polyglot Persistence：

```text
PostgreSQL      事务数据库
ClickHouse      分析数据库
Neo4j           图数据库
Redis           缓存数据库
MinIO           对象存储
Elasticsearch   搜索引擎
Vector DB       向量存储
```

# 4. 命名规范

---

## 表

格式：

```text
{domain}_{entity}
```

例如：

```text
identity_user

ontology_entity

workflow_task
```

---

## 主键

统一：

```sql
id UUID PRIMARY KEY
```

---

示例：

```sql
CREATE TABLE identity_user (

 id UUID PRIMARY KEY,

 username VARCHAR(100),

 created_at TIMESTAMP

);
```

---

## 外键

格式：

```text
{entity}_id
```

例如：

```text
user_id

role_id

workflow_id
```

# 14. ClickHouse模型

---

# fact_event

```sql
CREATE TABLE fact_event (

 event_time DateTime,

 event_type String,

 source String,

 payload String

)
ENGINE = MergeTree
ORDER BY (event_time,event_type);
```

---

# fact_agent_execution

```sql
CREATE TABLE fact_agent_execution (

 execution_id String,

 agent_code String,

 duration_ms UInt64,

 success UInt8

)
ENGINE = MergeTree
ORDER BY execution_id;
```

---

# fact_kpi

```sql
CREATE TABLE fact_kpi (

 snapshot_time DateTime,

 kpi_code String,

 value Float64

)
ENGINE = MergeTree
ORDER BY (snapshot_time,kpi_code);
```
