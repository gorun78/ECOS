# 1. 架构目标

ECOS不是单一业务系统。

其定位：

```text
Enterprise Cognitive Operating System

企业认知操作系统
```

需要同时满足：

```text
数据平台

语义平台

运营平台

Agent平台

认知平台
```

---

# 2. 架构原则

## P1 Domain Driven

所有设计围绕领域模型展开。

禁止：

```text
UserService
OrderService
```

式传统拆分。

采用：

```text
Identity Context

Semantic Context

Agent Context

Knowledge Context
```

拆分。

---

## P2 Ontology First

所有业务对象必须来自Ontology。

禁止：

```java
OrderEntity
CustomerEntity
```

直接耦合数据库。

必须：

```java
OntologyObject
```

统一抽象。

---

## P3 Event Driven

服务间通信优先事件。

禁止：

```text
同步级联调用
```

优先：

```text
Domain Event

Event Bus

Event Sourcing
```

---

## P4 AI Native

所有模块必须支持：

```text
Agent调用

Tool调用

语义检索
```
