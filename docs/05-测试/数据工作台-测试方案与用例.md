## 一、总体测试方案

### 1.1 测试目标

验证 ECOS 数据工作台 8 大模块的功能完整性、API 正确性、前端交互可用性、异常容错性，确保 Pipeline 2.0 全链路可正常运行。

### 1.2 测试范围

| 模块 | API 端点数 | 前端组件数 | 用例数 |
|------|:---------:|:---------:|:------:|
| 引擎配置 | 5 | 1 | ≥12 |
| Pipeline 任务 | 10 | 2 | ≥24 |
| PB 函数注册表 | 3 | 2 | ≥10 |
| UDF | 4 | 1 | ≥10 |
| Copilot | 3 | 1 | ≥8 |
| Git | 3 | 1 | ≥8 |
| 血缘 | 4 | 1 | ≥10 |
| 本体(跨界) | 2 | 1 | ≥6 |

### 1.3 环境要求

| 项目 | 要求 | 验证方式 |
|------|------|---------|
| Gateway | `http://localhost:8080` 启动正常 | `curl http://localhost:8080/actuator/health` |
| Vite Dev | `http://localhost:3000` 可用 | `curl http://localhost:3000/` |
| PostgreSQL | localhost:5432 连通 | Gateway 启动日志无 JDBC 错误 |
| Playwright | Node v22 + Chromium headless | `node -e "import('playwright').then(...)"` |
| 引擎状态 | 数据引擎已启动 | `GET /api/v1/engine/data/status` → STARTED |

### 1.4 测试策略

- **API 测试 (curl)**: 优先覆盖所有 API 端点，验证 HTTP 状态码、响应体结构、业务逻辑正确性
- **Web 测试 (Playwright headless)**: 覆盖前端交互场景 — 页面渲染、表单操作、拖拽连线、面板切换
- **链式场景**: 按"创建 → 编辑 → 执行 → 查看结果"完整链路验证
- **异常/边界**: 每个模块附加空参数、超时、404、权限、SQL注入等异常用例

### 1.5 优先级定义

| 优先级 | 含义 | 覆盖范围 |
|:------:|------|---------|
| **P0** | 核心功能，阻塞发布 | CRUD 主流程、引擎启停、Pipeline 执行 |
| **P1** | 重要功能，影响体验 | 搜索/筛选、Git 集成、配置刷新、Copilot |
| **P2** | 增强功能，可延后 | 列表空态、错误信息展示、非关键面板 |
