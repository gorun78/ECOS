## 二、目标状态

本体工作台应包含以下完整能力：

```
本体工作台 = 本体设计器 + 版本管理 + 工作流审批 + 质量规则 + Copilot + 图谱可视化
```

| 模块 | 目标 |
|------|------|
| **本体 CRUD** | 实体/属性/关系 CRUD 真实可用 |
| **引擎生命周期** | RUNNING/STOPPED 真实状态 + DB healthCheck + 启停端点 |
| **配置体系** | 本体引擎配置项 (图谱存储/缓存/版本策略) |
| **工作流** | buszhi 工作流纳入 ontology-engine 统一管理 |
| **版本管理** | 本体变更提案/审批/版本历史 |
| **图谱可视化** | FE 接真实 API (Neo4j/PgGraph)，不再用 Mock |
| **Copilot** | AI 辅助本体建模 |
| **Git 集成** | 本体定义存入 Git 仓库 |
| **前端集成** | 所有 FE 组件接真实 API, 不再使用 Mock |
